# RESOURCES.md format

Curated trusted sources. Draw lesson facts from here and cited links, not parametric guesses.

## Structure

```md
# {Topic} Resources

## Knowledge

- [Program outline](../../program/outline.md)
  Scope, audience, and block structure. Use for: mission alignment, section boundaries.
- [Session notes example](../../program/notes/example-block.md)
  Loads, reps, and cited claims for one block. Use for: factual drill lessons.

## Wisdom (Communities)

- [Coaches' forum or federation resource](https://example.com)
  When to use: practitioner perspective, not primary SoT.

## Gaps

- {Area mission needs but no trusted resource yet}
```

## Rules

- **High-trust only.** Project notes tied to sources, federation templates, primary references.
- **Annotate every entry.** One line: what it covers, when to reach for it.
- **Knowledge / Wisdom** groups mirror teach philosophy.
- **`## Gaps`** when mission needs coverage you lack.
- **Prune** wrong or shallow entries.
- **Record opt-out** if user declines communities.

For this repo: prefer `program/notes/`, `program/outline.md`, and `program/draft.md` over generic web summaries. Style craft: `docs/sports-communication-digest.md` (not factual). Source book (local): `data/Het Exacte Verhaal (clean).md`.
